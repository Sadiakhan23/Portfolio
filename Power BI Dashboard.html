Customer Churn Analytics Project

1. Problem Definition & Objective

  • Key Question: What factors are driving customer churn, and how can we predict and reduce churn to improve retention?
  • Business Importance: Churn impacts recurring revenue, customer lifetime value, and acquisition costs. Understanding churn helps prioritise effective retention strategies and optimise the customer experience.
  • Objectives/Hypotheses:
    • Identify key predictors of churn (e.g. tenure, support calls, payment delays).

2. Data Collection & Cleaning

  • Source: Kaggle
  • Data Type: Integer and String
  • Data Variables: CustomerID, Age, Gender, Tenure, Usage Frequency, Support Calls, Payment Delay, Subscription Type, Contract Length, Total Spend, Last Interaction, Churn
  • Cleaning Steps:
    • Checked for missing values in Total Spend, Last Interaction, and Payment Delay.
    • Encoded categorical variables using one-hot or label encoding.
    • Normalised numerical features like Usage Frequency and Total Spend.
    • Removed duplicates and ensured consistent data types.
  • Tools Used: Python (Pandas, NumPy)

3. Exploratory Data Analysis (EDA)

  • Initial Exploration: Summary statistics, distribution plots, and correlation matrices.
  • Key Variables Examined: Tenure, Usage Frequency, Support Calls, Payment Delay, Contract Length, Total Spend
  • Findings:
    • Short tenures and frequent support calls linked to higher churn.
    • Month-to-month contracts and delayed payments were strong churn indicators.
    • High Total Spend correlated with lower churn, suggesting value perception.
  • Visuals Created:
    • Churn rate by contract type and tenure.
    • Boxplots for spend vs churn.
    • Heatmap of feature correlations.
    • Power BI dashboard for stakeholder walkthrough.

4. Stakeholder Collaboration & Communication

  • Stakeholders: Marketing, Customer Success, Product, and Data Strategy teams.
  • Presentation Format:
    • Executive summary report with actionable insights.
    • Interactive dashboard (Power BI) for churn segmentation.
    • Slide deck for cross-functional review.
  • Communication Strategy:
    • Simplified churn drivers for non-technical teams.
    • Technical appendix for data science peers.
    • Recommendations tailored to departmental goals (e.g. retention campaigns, product tweaks).

5. Analytical Methods & Tools Used

  • Approach: K-Means clustering to segment customers by churn risk profiles.
  • Tools: Python (Scikit-learn, Matplotlib, Seaborn), Power BI
  • Rationale: Balanced interpretability and performance for actionable insights and stakeholder trust.
